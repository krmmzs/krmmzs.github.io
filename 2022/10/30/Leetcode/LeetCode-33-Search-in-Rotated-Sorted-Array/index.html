LeetCode 33. Search in Rotated Sorted Array

Main Idea

这个翻转让我想到一本书里关于三次reverse就可以得到旋转的方式

虽然和这题没有关系

You must write an algorithm with O(log n) runtime complexity.

给的数组升序且unique

如果不旋转直接二分log对吧。

翻转后可以得到两个升序数组, 不那么容易二分了

先找未知的旋转点，以nums[0]为界限，就可以二分出该未知点（二分的本质）

然后判断target在哪个区间，再二分

就可以判断出target是否存在
