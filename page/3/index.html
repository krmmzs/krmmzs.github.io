LeetCode 41. First Missing Positive

Posted on 2022-11-06

41. First Missing Positive

Main Idea

不能使用额外的空间，所以只能在原数组上进行操作。

不然哈希表就比较好解决了（hard的关键）

又要保证O(n)

能做的操作大概就是swap, 常数级的loop

这题看多点的案例会比较容易发现性质

[1, 2, 4, 5, ...] 的 first missing Positive is 3 [2, 3, 4, 5, ...] 的 first missing Positive is 1

可以发现答案是从1开始，如果1在正确的位置上那么就检查下一个数2,以此类推

考虑极端情况下，[1, 2, 3, 4, 5, ...] 的 first missing Positive is 6

所有的数都在正确的位置上，那么答案就是最后一个数+1（length + 1)

所以答案的范围是[1, length + 1], 其余数可以忽略不调整位置

Solution

Put each number in its right place.

For example:

When we find 5, then swap it with A[4].

At last, the first place where its number is not right, return the place + 1.

Job Control cheat

Posted on 2022-11-04 Edited on 2022-11-05 In Linux

Job Control cheat

Killing a process

  • Ctrl-C (SIGINT) - send SIGINT to the foreground process
  • Ctrl-\ (SIGQUIT) - send SIGQUIT to the foreground process
  • kill -TERM <PID> - send SIGTERM to the process with job number 1 A more generic signal for asking a process to exit gracefully

Pausing and backgrounding processes

  • jobs - list all jobs
  • pgrep <process> - list all processes with the name <process>
  • %<job> - the job with job number <job>
  • Ctrl-Z (SIGTSTP) - send SIGTSTP to the foreground process
  • fg - bring the most recently stopped process to the foreground
  • bg - bring the most recently stopped process to the background
  • & (suffix) - run a process in the background & suffix in a command will run the command in the background, giving you the prompt back, although it will still use the shell’s STDOUT which can be annoying (use shell redirections in that case).

Tips

让已经在运行的进程转到后台运行，您可以键入Ctrl-Z ，然后紧接着再输入bg 注意，后台的进程仍然是您的终端进程的子进程，一旦您关闭终端（会发送另外一个信号SIGHUP），这些后台的进程也会终止。 为了防止这种情况发生，您可以使用 nohup (一个用来忽略 SIGHUP 的封装) 来运行程序。 针对已经运行的程序，可以使用disown 或者使用tmux也行

很明显的优化是利用max的性质保证单调

现在这个队列，只要有后一个数比前一个数大，前一个数就没用了，可以直接丢弃。

好经典的一道题。

Z Library Possible Death

Z-Library

我为什么使用Z-Library

作为一个计算机的学生，应该迟早对国内的入门精通或者潦草的项目书籍感到厌烦。

就算可以阅读到很多中译书，也会对其翻译的质量感到不满。

那么你就可能需要一本英文原版的书籍。

国内的影印版:

amazon：

其他书:

我买不起。

盗版

Z-Library 当然是盗版

盗版，对于普通人来说是有好处，但对于出版社来说是灾难。

但是我不认可图书目前的商业模式。

有很多种方案可以优化，比如可以按照阅读时长或者引用次数付费。

互联网厉害的地方有: 我们可以通过一个链接到另一个链接，从一个知识到另一个知识。但书不行，我们必须通过买一本书后，笨拙的从一本书到另一本书，不能快速检索，验证和查看。

如果这本书有错别字怎么办？如果这本书有错误怎么办？如果只是读一个章节怎么办？如果只是想验证一下信息怎么办？

这一切都归因于——付费墙。

所以我说这不只是盗版的问题，而是商业模式的问题。亚马逊是最有可能做到世界图书馆的公司，但他们不会做，不符合它的利益。

自由当然不是免费，但一定可以有更好的解决办法可以做，而不只是现在想获得知识，先花钱。

在wiki 的右侧可以找到tor的链接。

然后你需要一个tor browser。

tor链接后，在tor browser中打开链接。
